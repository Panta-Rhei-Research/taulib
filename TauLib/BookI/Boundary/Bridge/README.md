# `TauLib/BookI/Boundary/Bridge/` — τ-native to Mathlib bridge modules

This directory hosts the **bridge layer** between τ-native
constructions (`TauInt`, `TauRat`, `TauReal`, `TauProfinite`,
ultrametric distance, cylinder topology, etc.) and Mathlib's
algebraic / topological / metric typeclass infrastructure.

The bridge modules are **canonical-anchored**: every typeclass
instance attaches to the τ-native carrier via a definitional
equality (or `rfl`-traceable computation) to the manuscript's
canonical structure. Verification handles (e.g.,
`dist_eq_ultrametricDistanceReal : dist x y = ultrametricDistanceReal x y := rfl`)
make the anchoring auditable.

## Workstream B1 — TauProfinite topology + metric + compactness (COMPLETE 2026-05-04)

The `TauProfinite*` modules in this directory implement manuscript
**Book II Part 2** chapters 9-10 (cylinder topology, ultrametric
depth) and **Part 3** chapter 14 (Topology Uniqueness). Together
they establish:

- **`MetricSpace TauProfinite`** anchored to the canonical
  `ultrametricDistance := 2^(-firstDisagreementDepth)` (manuscript
  II.D13)
- **`TopologicalSpace TauProfinite`** = the cylinder topology
  generated by `cylinderBasis` (Wave 50, manuscript II.D10)
- **`T2Space TauProfinite` + `TotallyDisconnectedSpace TauProfinite`**
  (Wave 51)
- **`CompactSpace TauProfinite`** (manuscript **II.T07**)
- The two topologies (cylinder + metric) are formally equal
  via two independent proof paths (manuscript **II.T10**)

algebraic τ-native carriers:

- **`TauInt*`** (`TauIntQuotient`, `TauIntQTransport`):
  `CommRing TauIntQ ≅ ℤ`
- **`TauRat*`** (`TauRatQuotient`, `TauRatQTransport`):
  `Field TauRatQ ≅ ℚ`
- **`TauReal*`** (`TauRealCongruence`, `TauRealQuotient`,

  `TauRealQuotientStrictOrderedRing`): full Cauchy-quotient

  `LinearOrderedField` is intentionally *not* shippable (Markov-
  principle / cardinality boundary); see
  `atlas/insights/2026-04-29-constructive-real-cardinality-boundary.md`.

## Discipline conventions

All modules in this directory follow the **strict-discipline 0-sorry
policy** enforced by CI (`scripts/check_no_sorry.py --root TauLib
--expected-axioms 3 --expected-sorry 0`):

- **0 `sorry`** — no temporary stubs; every theorem is fully proven
- **3 axioms** — exact count (only the manuscript's three foundational
  conjectural axioms; no new axioms introduced)
- **canonical-anchored instances** — every typeclass instance has
  a verification handle (a `rfl`-or-`decide`-provable equation
  tracing the instance back to the manuscript's canonical
  construction)
- **dual-path verification** for high-stakes anchoring decisions
  (e.g., `cylinder_topology_eq_metric_topology` is proven both via
  bidirectional inclusion AND via II.T10 uniqueness)
